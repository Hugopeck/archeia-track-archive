# Design: Archeia Plugin & Extensibility Strategy

Generated by /plan-ceo-review on 2026-03-22
Branch: Hugopeck/claude-extensibility | Mode: SCOPE EXPANSION

## Problem Statement

Archeia V0 ships as two standalone skill files that users manually copy into `.claude/skills/`. Claude Code now has a full plugin system with marketplace distribution, hooks for lifecycle automation, and bundled agents. The question: which extensibility primitives should Archeia leverage?

## Key Insight

**The plugin's value is distribution, not hooks.**

Claude Code's native CLAUDE.md/AGENTS.md loading already solves context injection — every session starts with architectural awareness because the docs are loaded automatically. Commit/PR-time enforcement (archeia check + GitHub Actions) is the right granularity for maintenance. Hooks are redundant with these existing primitives.

The plugin format solves the distribution problem: one-command install, marketplace discovery, versioning, team adoption.

## Chosen Approach: Dual-Format (Plugin + Standalone)

Ship Archeia as a Claude Code plugin AND maintain standalone skill files.

- Plugin is the primary distribution path (marketplace, `/plugin install`)
- Standalone skills remain for power users who prefer manual control
- Plugin and standalone are independent — a user installs one or the other

### Plugin Structure

```text
archeia-plugin/
├── .claude-plugin/
│   └── plugin.json          # name="archeia", version, description
├── skills/
│   ├── init/
│   │   ├── SKILL.md         # /archeia:init (was /archeia)
│   │   └── templates/       # same templates
│   └── ask/
│       └── SKILL.md         # /archeia:ask (was /archeia-ask)
└── hooks/
    └── hooks.json           # empty for now — ready for future use
```

### Standalone Structure (unchanged)

```text
.claude/skills/
├── archeia/
│   ├── SKILL.md             # /archeia
│   └── templates/
└── archeia-ask/
    └── SKILL.md             # /archeia-ask
```

## What We Evaluated and Skipped

### Hooks (all skipped)

| Hook | Why Skipped |
|------|-------------|
| SessionStart (context injection) | CLAUDE.md/AGENTS.md native loading covers this |
| PostToolUse (drift detection) | Per-edit is too noisy; commit/PR-time is right granularity |
| PostCompact (re-injection) | Native doc loading handles this |
| Stop (doc completeness) | Commit/PR-time enforcement sufficient (P4.12, F-005) |

### Custom Default Agent (skipped)

CLAUDE.md and AGENTS.md already serve as the context injection mechanism. A custom default agent is redundant and may conflict with other plugins.

### InstructionsLoaded Dynamic Enrichment (skipped)

Too speculative. Fires frequently. Performance risk.

## Distribution Impact

This design changes the P4 (Phase 1 Shipping) story:

- **P4.1** (distribution packaging): Plugin format replaces "git clone + compiled Bun binary"
- **P4.3** (Cursor marketplace): Plugin format may apply to Cursor's plugin system
- **P4.4** (skill publishing): Becomes `/plugin install archeia@marketplace`
- **New TODO**: Submit to official Anthropic marketplace after plugin is built

## Success Criteria

1. Plugin installs cleanly with `--plugin-dir` (local testing)
2. `/archeia:init` and `/archeia:ask` are registered and functional
3. Templates resolve correctly from plugin-installed location
4. Standalone skills still work independently
5. TODO.md reflects the plugin distribution strategy
