# Preferences

These are tie-breakers for when multiple choices are valid.

## Product Preferences

- Prefer skill instructions over compiled binaries.
- Prefer templates over larger generation pipelines.
- Prefer the agent's native capabilities over custom tooling.
- Prefer stronger instructions and templates over new workflow machinery when capable agents can already do the work.
- Prefer readable, opinionated markdown projections over generic generated status dumps.
- Prefer pull-request confirmation over custom approval flows.
- Prefer maintaining a few critical docs well over generating many shallow docs.

## Writing Preferences

- Prefer explicit over clever.
- Prefer short agent-facing files that are loaded often.
- Prefer evidence-backed wording over confident abstraction.
- Prefer repo-specific language over reusable but generic prose.
- Prefer updating docs in the same change as the behavior they describe.

## Workflow Preferences

- Prefer inline maintenance when the change is obvious.
- Prefer drift-to-PR when the wording deserves review.
- Prefer GitHub Action support as optional infrastructure, not a prerequisite.
- Prefer migration of good existing docs over replacing them wholesale.

## Business Preferences

- Prefer 5 real users over 1,000 theoretical installs.
- Prefer retention evidence over growth narratives.
- Prefer honest uncertainty over premature monetization stories.
- Prefer distribution through existing agent ecosystems and git workflows.

## Repo Preferences

- Prefer keeping historical code visible rather than pretending it never existed.
- Prefer small focused Markdown files over monolithic docs.
- Prefer changes that keep the repo understandable to future agents and humans.
