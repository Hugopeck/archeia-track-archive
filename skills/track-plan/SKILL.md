# Track Plan

## Overview

Plan a large initiative — define scope, phases, and sequence — then create tasks via track-decompose.

## Workflow

1. Read `.track/PROTOCOL.md` if you haven't already this session.
2. Understand the project goal.
3. Add project to `.track/config.yaml` with title and description.
4. Scope: what's in, what's out, constraints, definition of done.
5. Phase if large: foundation → core → integration → testing.
6. Decompose each phase using track-decompose workflow.
7. Present dependency graph before creating tasks.

## Response

Report: project key/title, phases, task count, max parallelism, dependency graph.
